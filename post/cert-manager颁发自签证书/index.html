cert-manager简介

cert-manager 是一个云原生证书管理开源项目，用于在 Kubernetes 集群中自动管理和颁发来自各种颁发源的 TLS 证书，它可以从各种受支持的来源颁发证书，包括 Let’s Encrypt、HashiCorp Vault和Venafi以及私有 PKI，它将确保证书定期有效和更新，并在到期前的适当时间尝试更新证书。

apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: kuard
  annotations:
    cert-manager.io/cluster-issuer: "cluster-ca-issuer"
spec:
  ingressClassName: nginx
  tls:
  - hosts:
    - kuard.nblh.local
    - w1.nblh.local
    secretName: kuard-tls
  rules:
  - host: kuard.nblh.local
    http:
      paths:
      - path: /
        pathType: Prefix
        backend:
          service:
            name: kuard
            port:
              number: 80

test           kuard-tls       True    kuard-tls              3s

手动更新证书

1
2
3
4
5
6
7
8

kubectl edit certificate kuard-tls -n test
# 在spec字段下添加以下两行
  duration: 87600h # 10 years
  renewBefore: 720h # 30 days
# 保存并退出

# 删除原来的secret，让certmanager重新自动生成secret
kubectl get certificate kuard-tls -o=jsonpath='{.spec.secretName}' | xargs kubectl delete secret
